Here you will find details about a few of the major applications I have written whilst working for Pinpointers.

Pinpointers Web Application

  • ExtJS 3
  • HTML5
  • CSS
  • Google Maps API v3
  • AJAX/WebSocket
  • SQL Reporting Services

The primary web application used by customers for locating vehicles in their fleet fitted with Pinpointers tracking devices.

Users are able to view current and historial position data, including speed and heading. The application makes use of AJAX requests, and also displays information on a Google Maps display.

A couple of examples of the many cool features I have developed for this web application can be found below.

Location Search Tool

Users needed to be able to search addresses and view on a Google map within the main application.

With the Google Maps API already in use, I decided to use the Geocoder service within the API.

Whilst this would provide users with the ability to search address, I also wanted the search tool to allow users to search across placemarkers (points of interests) stored in the database against their login account.

Find Nearest Vehicle

Typically customers use the web application for judging how far a vehicle is from a location of interest, such as their own customer sites. We needed to give users the ability to determine which of their vehicles was the closest to a given location and what the likely travel time would be.

Using Googles Directions service was the most obvious choice here since this is available as part of their Maps API already in use within the application.

Coach Manager Integration

  • Coach Manager
  • ExtJS 4
  • Sencha Touch 2
  • Google Maps API v3
  • AJAX/WebSocket/FTP
  • KML/XML
  • Secuirty Tokens

Pinpointers provide vehicle tracking solutions for small to medium sized bus and coach operators, serving local school contracts and offer public transport routes.

Many of these customers use Coach Manager, a contract and private hire booking system provided by Distinctive Systems.

Pinpointers Jobs

Through discussions with the provider, Pinpointers have been able to receive a daily XML feed of booking information sent over FTP. Using this data, Jobs are created in the database detailing the start and stop locations, expected arrival and departure times and allocated vehicle information.

With the Coach Manager information loaded, the Pinpointers database is able to use the live tracking data for allocated vehicles across the created jobs to provide real time status updates, such as whether the vehicle is running on time or has missed an expected departure.

Public Web Pages

Using the Coach Manager Job data discussed in the previous project example, I have developed a web application displaying allocated vehicle locations on a Google Map. Vehicles are dynamically shown and hidden whenever they start and finish operating on their allocated contracts.

I also added route overlays to the map display to display customer routes and stops visited. This was achieved using KML data, typically exported using Google MyMaps, and imported via the Google Maps API.
